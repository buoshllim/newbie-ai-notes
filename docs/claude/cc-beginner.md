# Claude Code 설치 및 초기 세팅

Claude Code는 데스크탑 앱으로도 대부분의 작업을 처리할 수 있음. 터미널 버전이 꼭 필요한 건 아니지만, 나중에 다른 자동화 도구와 연계해서 쓰거나, 하네스 엔지니어링을 해보고 싶을 때, 코드로 직접 제어하고 싶을 때 등을 대비해 세팅 과정을 기록해 봄.

---

## 유료 플랜 필요함

Pro나 Max 플랜이 있어야 함.

| 플랜 | 가격 | 비고 |
|---|---|---|
| Claude Pro | $20 | 현재는 사용 가능. 일부 신규 가입자는 안 될 수도 있음 |
| Claude Max | $100 / $200 | 확실히 됨 |

2026년 4월 24일 기준으로 기존 Pro 사용자도 사용할 수 있는데, 언제 Max만 쓸 수 있게 바뀔지 모름.

## 설치

### macOS
터미널 열고 아래 붙여넣기. 너무 쉽게 끝남.

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

### Windows
윈도우는 다소 어려움. 아래 순서대로 잘 따라해야함.

시작 버튼 → `PowerShell` 검색 → 마우스 오른쪽 클릭 → `관리자 권한으로 실행`

창이 열리면 먼저 아래 입력:

```powershell
cd\
```

`cd\`는 C:\ 드라이브 최상위 폴더로 이동하는 명령어. 관리자 PowerShell은 시작 위치가 제각각이라 헷갈릴 수 있어서 일단 여기서 시작하는 게 편함. 설치 자체는 어느 폴더에서 해도 상관없으니 익숙해지면 원하는 폴더에서 해도 됨.

그 다음 설치 명령어를 실행(복사 붙여넣기):

```powershell
irm https://claude.ai/install.ps1 | iex
```

> **회사에서는 안 될 수도 있음.** 그럴 땐:
> - 집에서 개인 PC로 해보거나
> - 회사 노트북으로 외부망(집)에서 해보거나
> - 가장 추천하는 건 **맥 미니**. 항상 켜두고 원격으로도 쓸 수 있고, Claude Code 외에도 활용처가 다양함.

> 혹시 이 명령어로 설치가 안 되면 Claude 웹/앱에서 "Node.js 기반으로 설치하는 방법 알려줘"라고 물어보고 알려주는 대로 차근차근 해보면 됨.


### Windows PATH 설정 (설치 후 필수)

윈도우에서는 설치가 완료돼도 `claude` 입력하면 실행이 안 되는 경우가 있음. PATH가 안 잡혀서 실행이 안된다는 메시지가 나옴. 그럼 아래 순서대로 진행하면 됨.

키보드에서 `윈도우 키 + R` 누르기 → 뜨는 실행창에 아래 입력하고 Enter:

```powershell
sysdm.cpl
```

`시스템 속성` 창이 뜨면, `고급` 탭 클릭 → 맨 아래 `환경 변수` 버튼 클릭

아래쪽 `시스템 변수` 목록에서 `Path` 찾아서 더블클릭 → `새로 만들기` 버튼 → 아래 경로 붙여넣기:

```powershell
C:\Users\내계정이름\.local\bin
```

> `내계정이름` 부분은 본인 Windows 계정 이름으로 바꿔야 함. 설치 완료 화면에 `Location: C:\Users\???\.local\bin\claude.exe` 식으로 경로가 표시되니까 거기서 확인하면 됨.

`확인` → `확인` → PowerShell 완전히 닫고 새로 열기 → `cd\` 입력 → `claude --version` 입력해서 버전 번호가 보이면 성공.

## Cladue Code 실행

터미널(macOS/Linux) 또는 PowerShell(Windows)을 열고 아래 명령어를 입력.
> 윈도우에서는 `cd\` 입력 후에 아래 명령어를 입력

```bash
claude
```

처음 실행하면, `Mode` 선택하는 화면이 나오는데 마음에 드는 것을 선택하면 됨(ex. `Dark Mode`). 그리고 브라우저가 자동으로 열리면서 `Anthropic 로그인` 화면이 나옴. `계정으로 로그인`하면 `터미널`로 돌아오면 됨. 이후엔 자동 연결이라 다시 로그인 안 해도 됨. `Safety Check` 선택 화면에서는 `1. Yes, I trust this folder` 선택하면 됨.

## CLAUDE.md — Claude한테 나를 소개하는 파일

Claude는 대화를 시작할 때마다 내가 누군지 모름. 매번 "나 이런 사람이야, 이렇게 답해줘" 설명해야 하는데, CLAUDE.md에 써두면 알아서 읽고 시작함.

한 번 써두면 Claude Code 켤 때마다 자동으로 적용됨.

담을 수 있는 내용 예시:
- **대화 방식** — "반말로 편하게 얘기해줘", "요약은 3줄 이내로"
- **내 정보** — 직업, 관심사, 지금 배우고 있는 것
- **환경** — 쓰는 OS, 자주 쓰는 도구들
- **성격 정의** — Claude를 어떤 캐릭터로 쓰고 싶은지. 예를 들어 "친한 친구처럼 편하게", "냉정하게 팩트만" 같은 식으로 지정하면 그 톤으로 일관되게 대화함

### 만드는 법 — Claude Code랑 대화하면서 만들면 됨

Claude Code를 켜고 이렇게 말해보면 됨:

```
나한테 맞는 CLAUDE.md 파일 만들어줘.
나는 [직업/역할]이고, [관심사/현재 하고 있는 것]에 관심이 있어.
대화할 때는 [원하는 스타일]로 대답해줘.
```

그러면 Claude가 직접 파일을 만들고 저장해줌. 나중에 "CLAUDE.md에 이것도 추가해줘" 하면 바로 수정도 해줌.

### 적용 범위
`~/.claude/CLAUDE.md` — 어느 폴더에서 실행해도 항상 적용됨.

나중에 특정 프로젝트에만 다른 설정을 쓰고 싶어지면 그 폴더 안에 `CLAUDE.md`를 따로 만들면 됨. 처음에는 전체 설정 하나면 충분함.

## 모델 확인 및 변경

기본값은 Sonnet. 처음엔 그냥 써도 됨.

현재 어떤 모델인지 확인하려면:

```
/model
```

이런 화면이 뜸:

```
❯ 1. Default (recommended) ✔  Sonnet 4.6 · Best for everyday tasks
  2. Opus                     Opus 4.7 · Most capable for complex work · ~2× usage vs Sonnet
  3. Haiku                    Haiku 4.5 · Fastest for quick answers
```

방향키로 선택하고 Enter 하면 바뀜.

- **Sonnet** — 기본값. 일반적인 용도엔 이걸로 충분함
- **Opus** — 더 복잡한 작업에 적합. 사용량 소모가 Sonnet의 2배
- **Haiku** — 빠르고 가벼움. 단순한 질문에 적합

## 하단 상태바 (statusline)

터미널 맨 아래에 현재 모델, 컨텍스트 사용률, 플랜 사용 한도를 표시해 줌.

아래 프롬프트 그대로 복사해서 붙여넣기 하면 설정됨.

```
statusline을 설정해줘. 표시 항목:
- 모델명
- context:N% (현재 대화 컨텍스트 창 사용률)
- 5h:N% 7d:N% (플랜 사용 한도)
```

이렇게 뜸:
```
Sonnet 4.6 | context:42% | 5h:18% 7d:34%
```

| 항목 | 의미 |
|---|---|
| `context:N%` | 현재 대화 맥락이 얼마나 찼는지 |
| `5h:N%` | 최근 5시간 플랜 사용량 |
| `7d:N%` | 최근 7일 플랜 사용량 |

`context`가 70~80% 넘어가면 `/compact` 입력하는 게 좋음. Claude가 대화 내용을 요약해서 맥락을 압축해줌. 안 하면 대화가 길어질수록 앞 내용을 잊어버리거나 응답이 느려질 수 있음. 한 세션에서 오래 작업할 때 특히 필요함.

## Claude Code 나가는 법

터미널이나 PowerShell 창을 그냥 닫으면 됨. 가장 간단한 방법.

창은 유지하고 Claude Code만 나와서 일반 터미널로 돌아가고 싶을 때는:

```
/exit
```

차이:
- **창 닫기** — 터미널 자체를 끔. 다시 쓰려면 터미널 새로 열어야 함
- **`/exit`** — Claude Code만 종료. 터미널은 그대로 남아서 다른 명령어 바로 입력 가능

파일 복사, 폴더 이동 같은 일반 터미널 작업을 Claude Code 끄고 해야 할 때 `/exit` 씀. 아니면 그냥 창 닫아도 됨.

claude

## 직접 뭔가 만들어보기

간단한 걸 하나 만들어보면 훨씬 빨리 익숙해 짐.

**내 소개 페이지 만들기**

1. Claude Code 켜기

2. 아래 프롬프트 복사해서 붙여넣기:

   ```
   my-page 폴더 만들어줘.
   ```

   Claude가 폴더를 만들어줌. `my-page`는 자유롭게 변경.

3. 만들었다고 하면, 이어서 아래 프롬프트 붙여넣기 (괄호 안은 직접 채워서):

   ```
   방금 만든 폴더 안에 내 소개 페이지 HTML 파일 만들어줘. 궁금한 내용은 나한테 물어보고 만들어.
   ```

4. `my-page` 폴더 → `index.html` 더블클릭해서 브라우저로 열어보기

마음에 안 드는 부분은 "배경색 바꿔줘", "폰트 더 크게" 식으로 말하면 바로 수정할 수 있음.

## 부록

### 자주 쓰는 명령어

Claude Code세션 안에서 씀.

| 명령어 | 용도 |
|---|---|
| `/clear` | 대화 초기화. 새 주제 시작할 때 |
| `/compact` | 대화 길어지면 압축 |
| `/model` | 모델 확인·변경 |
| `Ctrl+C` | 답변 중간에 멈추기 |

### 퍼미션 없이 실행하기

Claude Code는 기본적으로 파일 수정이나 명령어 실행 시 매번 허락을 물어봄. 익숙해지면 이 확인 과정이 번거롭게 느껴질 수 있음.

처음에 Claude 실행할 때 아래처럼 입력하고 실행하면 퍼미션 확인 없이 진행함. (그래도 물어보는 것이 있긴 함.)

```bash
claude --dangerously-skip-permissions
```

"dangerously(위험하게)"가 붙어있는 이유는, Claude가 실수로 파일을 삭제하거나 의도치 않은 명령을 실행해도 중간에 막을 수 없기 때문. Claude Code에 익숙해지고 신뢰할 수 있을 때 사용하는 게 좋음.

---

### 텔레그램/디스코드 연동

Claude Code는 텔레그램이나 디스코드와 연동해서 채팅으로 명령하거나, 모바일에서 원격으로 접속하는 것도 가능함. 이 부분은 추후 업데이트 예정.


### Claude Code 자체 업데이트

Cladue Code 자체도 업데이트가 필요한데, 어렵게 생각할 필요 없이 그냥 Claude Code에게 물어보면 됨.

```
Claude Code 최신 버전으로 업데이트해줘
```

직접 업데이트 안해주고 방법을 알려주면 그대로 따라해도 되고, Claude에게 직접 해달라고 다시 요청하면 됨. 보통은 직접 해줌.

### IDE(코드 편집기) 같이 쓰면 편함

Claude Code는 터미널만으로도 돌아가지만, IDE에서 내장 터미널을 열고 Claude Code를 같이 쓰면 파일 수정 내용을 눈으로 보면서 작업할 수 있어서 훨씬 편해짐. 코딩을 안 해도 파일, 마크다운 문서 편집 용도로도 충분히 씀.

**VS Code** — [code.visualstudio.com](https://code.visualstudio.com)

- 무료, 비교적 가볍고 안정적
- Claude Code 공식 확장 프로그램 있음. 설치하면 터미널 열지 않고 VS Code 안에서 바로 실행 가능
- 로그인 불필요. 회사 환경에서도 별다른 문제 없이 돌아감

**Antigravity** — Google이 만든 IDE. VS Code 기반

- Gemini 모델 내장. Claude 모델로 바꿔서 쓸 수도 있음
- 회사에서는 설치 후 Google 로그인 리다이렉트 과정에서 문제가 발생하여 사용하지 못하는 경우가 있음.
- 2026년 기준 아직 안정성 이슈 있다는 후기가 있어, 일단은 VS Code로 시작하는 것을 추천.
