# Claude Code 처음 설치하고 세팅하기

터미널에서 돌아가는 AI. 쓰다보면 GUI보다 더 편함.  
파일도 읽고, 코드도 짜고, 명령어도 실행함. 원격 조종이 쉬워서 자주 쓰게 됨.

---

## 유료 플랜 필요함

무료로는 안 됨. Pro나 Max 중 하나 있어야 함.

| 플랜 | 월 비용 | 비고 |
|---|---|---|
| Claude Pro | $20 | 처음엔 이걸로 충분함 |
| Claude Max | $100~200 | 많이 쓸 것 같으면 |

## 설치

### macOS / Linux
터미널 열고 아래 붙여넣기.

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

### Windows
시작 버튼 → `PowerShell` 검색 → 마우스 오른쪽 클릭 → **관리자 권한으로 실행**

창이 열리면 먼저 아래 입력:

```powershell
cd\
```

`cd\`는 C:\ 드라이브 최상위 폴더로 이동하는 명령어. 관리자 PowerShell은 시작 위치가 제각각이라 헷갈릴 수 있어서 일단 여기서 시작하는 게 편함. 설치 자체는 어느 폴더에서 해도 상관없으니 익숙해지면 원하는 폴더에서 해도 됨.

그 다음 설치 명령어 실행:

```powershell
irm https://claude.ai/install.ps1 | iex
```

> 혹시 설치가 안 되면 Claude 웹/앱에서 "Node.js 기반으로 설치하는 방법 알려줘"라고 물어보면 됨.

> **회사 PC에서 안 될 수도 있음.** 사내망이나 보안 정책으로 막혀있는 경우가 많음. 그럴 땐:
> - 집에서 개인 PC로 해보거나
> - 회사 노트북을 집에 가져가서 해보거나
> - 가장 추천하는 건 **맥 미니**. 항상 켜두고 원격으로도 쓸 수 있고, Claude Code 말고도 활용처가 무궁무진함.

### 설치 확인
```bash
claude --version
```

버전 번호 뜨면 설치 완료!

## 로그인

터미널(macOS/Linux) 또는 PowerShell(Windows)을 열고 아래 입력.

```bash
claude
```

처음 실행하면 브라우저가 자동으로 열리면서 Anthropic 로그인 화면이 나옴. 계정으로 로그인하면 터미널로 돌아오고 바로 쓸 수 있음. 이후엔 자동 연결이라 다시 로그인 안 해도 됨.

## CLAUDE.md — Claude한테 나를 소개하는 파일

Claude는 대화를 시작할 때마다 내가 누군지 모름. 매번 "나 이런 사람이야, 이렇게 답해줘" 설명해야 하는데, CLAUDE.md에 써두면 알아서 읽고 시작함.

한 번 써두면 Claude Code 켤 때마다 자동으로 적용됨.

담을 수 있는 내용 예시:
- **대화 방식** — "반말로 편하게 얘기해줘", "요약은 3줄 이내로"
- **내 정보** — 직업, 관심사, 지금 배우고 있는 것
- **환경** — 쓰는 OS, 자주 쓰는 도구들
- **성격 정의** — Claude를 어떤 캐릭터로 쓰고 싶은지. 예를 들어 "친한 친구처럼 편하게", "냉정하게 팩트만" 같은 식으로 지정하면 그 톤으로 일관되게 대화함

### 만드는 법 — Claude Code랑 대화하면서 만들면 됨

Claude Code를 켜고 이렇게 말해보면 됨:

```
나한테 맞는 CLAUDE.md 파일 만들어줘.
나는 [직업/역할]이고, [관심사/현재 하고 있는 것]야.
대화할 때 [원하는 스타일]로 해줬으면 해.
```

그러면 Claude가 직접 파일을 만들고 저장해줌. 나중에 "CLAUDE.md에 이것도 추가해줘" 하면 바로 수정도 해줌.

### 적용 범위
`~/.claude/CLAUDE.md` — 어느 폴더에서 실행해도 항상 적용됨.

나중에 특정 프로젝트에만 다른 설정을 쓰고 싶어지면 그 폴더 안에 `CLAUDE.md`를 따로 만들면 됨. 지금은 전체 설정 하나면 충분함.

## 모델 확인 및 변경

기본값은 Sonnet. 처음엔 그냥 써도 됨.

현재 어떤 모델인지 확인하려면:

```
/model
```

이런 화면이 뜸:

```
❯ 1. Default (recommended) ✔  Sonnet 4.6 · Best for everyday tasks
  2. Opus                     Opus 4.7 · Most capable for complex work · ~2× usage vs Sonnet
  3. Haiku                    Haiku 4.5 · Fastest for quick answers
```

방향키로 선택하고 Enter 하면 바뀜.

- **Sonnet** — 기본값. 일반적인 용도엔 이걸로 충분함
- **Opus** — 더 복잡한 작업에 적합. 사용량 소모가 Sonnet의 2배
- **Haiku** — 빠르고 가벼움. 단순한 질문에 적합

## 하단 상태바 (statusline)

터미널 맨 아래에 현재 모델, 컨텍스트 사용률, 플랜 한도 보여줌. 아래 프롬프트 그대로 붙여넣으면 설정됨.

```
statusline을 설정해줘. 표시 항목:
- 모델명
- context:N% (현재 대화 컨텍스트 창 사용률)
- 5h:N% 7d:N% (플랜 사용 한도)
```

이렇게 뜸:
```
Claude Sonnet 4.6 | context:42% | 5h:18% 7d:34%
```

| 항목 | 의미 |
|---|---|
| `context:N%` | 현재 대화 맥락이 얼마나 찼는지 |
| `5h:N%` | 최근 5시간 플랜 사용량 |
| `7d:N%` | 최근 7일 플랜 사용량 |

`context`가 70~80% 넘어가면 `/compact` 입력하는 게 좋음. Claude가 대화 내용을 요약해서 맥락을 압축해줌. 안 하면 대화가 길어질수록 앞 내용을 잊어버리거나 응답이 느려질 수 있음. 한 세션에서 오래 작업할 때 특히 필요함.

## Claude Code 나가는 법

터미널이나 PowerShell 창을 그냥 닫으면 됨. 가장 간단한 방법.

창은 유지하고 Claude Code만 나와서 일반 터미널로 돌아가고 싶을 때는:

```
/exit
```

차이:
- **창 닫기** — 터미널 자체를 끔. 다시 쓰려면 터미널 새로 열어야 함
- **`/exit`** — Claude Code만 종료. 터미널은 그대로 남아서 다른 명령어 바로 입력 가능

파일 복사, 폴더 이동 같은 일반 터미널 작업을 Claude Code 끄고 해야 할 때 `/exit` 씀. 아니면 그냥 창 닫아도 됨.

## 직접 뭔가 만들어보기

간단한 걸 하나 만들어보면 Claude Code가 뭔지 훨씬 빨리 감이 잡힘.

**내 소개 페이지 만들기**

1. 빈 폴더 하나 만들기 (이름은 자유. 예: `my-page`)
2. 그 폴더에서 Claude Code 실행

   macOS라면:
   ```bash
   cd ~/Desktop/my-page
   claude
   ```

3. Claude Code에 아래 프롬프트 붙여넣기 (괄호 안은 직접 채워서):

   ```
   내 소개 페이지 HTML 파일 만들어줘.
   이름: [이름]
   직업 또는 관심사: [내용]
   오늘 한 일: [내용]
   디자인: 심플하고 다크 테마, 한국어
   ```

4. 폴더 안에 `index.html` 파일이 생기면 더블클릭해서 브라우저로 열어보기

마음에 안 드는 부분은 "배경색 바꿔줘", "폰트 더 크게" 식으로 말하면 바로 수정해줌.

## 텔레그램/디스코드 연동

Claude Code를 텔레그램이나 디스코드와 연동해서 채팅으로 명령하거나, 모바일에서 원격으로 접속하는 것도 가능함. 이 부분은 따로 정리해서 업데이트 예정.

## 부록

### 자주 쓰는 명령어

Claude Code세션 안에서 씀.

| 명령어 | 용도 |
|---|---|
| `/clear` | 대화 초기화. 새 주제 시작할 때 |
| `/compact` | 대화 길어지면 압축 |
| `/model` | 모델 확인·변경 |
| `Ctrl+C` | 답변 중간에 멈추기 |

### Claude Code 업데이트

업데이트 방법이 헷갈리면 Claude Code에게 물어보면 됨.

```
Claude Code 최신 버전으로 업데이트하는 방법 알려줘
```

OS에 맞게 안내해줌.

---

### VS Code가 있으면

Visual Studio Code(VS Code)를 설치해두면 Claude Code와 함께 쓸 때 편해짐. 터미널 없이 VS Code 안에서 바로 Claude Code를 실행할 수 있고, 파일 수정 내용을 나란히 보면서 작업할 수 있음. 코드 에디터지만 코딩 안 해도 파일 관리나 마크다운 편집 용도로도 충분히 씀.

[code.visualstudio.com](https://code.visualstudio.com) 에서 무료로 받을 수 있음.
